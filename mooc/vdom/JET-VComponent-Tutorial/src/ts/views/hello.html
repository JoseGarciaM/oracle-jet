Hello, World!

Let's jump in and take a look at our first VComponent: 

    
oj-hello-world.tsx:

    
  1    import { h, Component } from 'preact';
  2    import { customElement, GlobalProps } from 'ojs/ojvcomponent';
  3
  4    @customElement('oj-hello-world')
  5    export class HelloWorld extends Component<GlobalProps> {
  6      render() {
  7        return <div>Hello, World!</div>;
  8      }
  9    }

Some points worth noting about the above code:

  • VComponents are defined as Typescript modules in files with the extension ".tsx".
  • VComponents extend the Preact Component base class (line #5), which is imported directly from the preact module (line #1).
  • All VComponents minimally support the set of global HTML attributes defined by the GlobalProps type. If the VComponent has no other component-specific properties, we use GlobalProps for the Component's first type parameter (line #5)
  • All VComponents must define a render() method (line #6).
  • The render() method is responsible for returning a virtual DOM tree that represents the VComponent content (line #7).
  • The virtual DOM tree is expressed via JSX elements (line #7), which is an extension to the EcmaScript/TypeScript languages for embedding markup in code.
  • The custom element tag name ('oj-hello-world') is specified via the @customElement decorator (line #4), which is imported from the ojs/ojvcomponent module (line #2).
  • For the moment, we manually import the virtual DOM node factory function, named "h" (line #1). At build time, JSX elements are translated into calls to this function. (We expect the requirement to manually import the h function to be removed in a future release.)

Once the VComponent has been defined, built and imported into the app, it can be used like any other custom element. For example, embedding the following code in an HTML page or template (eg. index.html, or a composite component or oj-module view template): 

    
      <!-- Just a plain old HTML custom element -->
      <oj-hello-world></oj-hello-world>

One small step that we glossed over: in order for this to work, the oj-hello-world.tsx module needs to be imported into the app. This is accomplished by including an import of the form: 

    
      import "oj-hello-world/loader";

Somewhere in the app, such as in a view model.

If you have experience authoring JET composite components, the use of a "loader" module in the above import should be familiar. We explore the role of this module in the next topic.
